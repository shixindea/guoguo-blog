# 完善文章列表与侧边栏逻辑

## 后端修改 (Backend)
1.  **修复标签筛选 Bug**: 
    - 目前 `ArticleServiceImpl` 中的标签筛选是在内存中进行的（先查分页再过滤），会导致分页数据不准确。
    - **修复方案**: 改为在 `ArticleRepository` / `Specification` 层使用 JPA `Join` 进行数据库级筛选。
2.  **增强排序支持**:
    - 确保 `list` 接口支持按 `commentCount` (评论数) 排序，用于实现“本周热议”。

## 前端修改 (Frontend)
1.  **推荐侧边栏 (`RecommendSidebar.tsx`)**:
    - **热门标签**: 对接 `GET /api/tags/popular` 接口。
    - **本周热议**: 对接 `GET /api/articles` 接口（参数 `sortBy=commentCount&order=desc`）。
    - **24h 飙升榜**: 暂时对接 `GET /api/articles/trending`（浏览量榜单）。
2.  **文章列表页 (`ArticleList.tsx` 及父组件)**:
    - **分类筛选**: 新增 `CategoryFilter` 组件，对接 `GET /api/categories`，支持点击分类筛选文章。
    - **排序切换**: 实现“热门推荐”(sortBy=viewCount) 与 “最新发布”(sortBy=publishedAt) 的切换逻辑。
    - **申请标签**: 暂时实现为提示或跳转到反馈页。

## 执行顺序
1.  修改后端 `ArticleServiceImpl` 与 `ArticleRepository`，修复 Tag 筛选并确认排序字段。
2.  运行后端测试确保接口逻辑正确。
3.  修改前端 `RecommendSidebar` 组件对接数据。
4.  修改前端文章列表页面，增加分类筛选与排序切换。
5.  验证前后端联调效果。
